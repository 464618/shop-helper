# 电商 ERP 业务学习驾驶舱

这是一个本地优先 PWA 学习工具，项目英文名称为 `ecommerce-erp-business-learning-pwa`。它面向电商 ERP 管理员，帮助你沉淀电商术语、ERP 字段、业务流程、数据指标、运营动作、业务场景、练习题和学习笔记。

## 适合谁使用

适合电商 ERP 系统管理员、想转向数据分析岗位的业务同学，以及需要理解订单、库存、售后、投流、毛利、退款等数据的人。

## 为什么适合 ERP 管理员转数据分析师

ERP 管理员通常熟悉系统操作，但容易缺少业务口径、指标逻辑和分析框架。本项目把“字段 - 流程 - 指标 - 运营动作 - 业务场景”放在一起，方便你把每天看到的数据和业务动作联系起来。

## 为什么不做登录注册、数据库、爬虫、真实 ERP 接口和付费 API

这是个人本地学习工具，不需要账号体系。当前数据量小，使用 JSON 示例数据和 localStorage 已经足够。项目不抓取真实平台数据，不连接公司 ERP，不读取公司真实数据，不接入 OpenAI、DeepSeek 或任何需要 API Key 的服务。

## 本地运行方法

方式 1：使用 VS Code 免费插件 Live Server 打开项目目录。

方式 2：如果电脑有 Python，可以在项目目录运行：

```bash
python -m http.server
```

然后用浏览器打开对应地址，通常是 `http://localhost:8000`。直接双击 `index.html` 可能会导致 JSON 加载失败，建议用 Live Server 或 Python 本地服务器运行。

## 每个模块功能说明

- 首页：展示统计、今日建议学习内容、学习阶段和备份提醒。
- 电商术语库：搜索、分类筛选、新增、删除本地新增、重置本地新增。
- ERP 字段解释器：理解字段业务含义，支持搜索、筛选和本地新增。
- 业务流程地图：展示商品、订单、库存、售后、运营、财务、物流流程。
- 指标口径库：理解指标公式、字段来源和常见口径坑。
- 运营动作库：理解降价、投流、补货、清仓等动作影响。
- 业务场景分析库：训练从数据异常倒推业务原因。
- 业务练习题：查看答案、学习解析、标记已掌握。
- 指标计算器：本地计算毛利、毛利率、退款率、客单价、ROI、净销售额、成本率。
- 我的学习笔记：记录每天遇到的不懂内容。
- 数据备份与同步：导出、导入、清空本地用户数据。

## 数据文件和 localStorage 说明

`data/` 目录中的 JSON 都是模拟学习数据，不是真实公司数据。用户新增内容保存在浏览器本地 localStorage 中，包括术语、ERP 字段、指标、运营动作、业务场景、学习笔记和已掌握题目。清理浏览器数据可能会删除这些内容，所以建议定期导出备份。

## PWA 说明

PWA 是 Progressive Web App，意思是网页可以像小软件一样被安装到桌面或手机主屏幕，并支持基础离线访问。PWA 安装能力可能受浏览器和 HTTPS 限制。本地开发时，`localhost` 通常可以测试基础 PWA 能力。

## 电脑和手机使用

电脑上用 Live Server 或 `python -m http.server` 启动项目，然后在浏览器中打开地址。手机和电脑在同一 Wi-Fi 下时，可以用电脑局域网 IP 地址访问本地服务器，并在手机浏览器菜单中选择“添加到主屏幕”。本项目不负责公网部署。

## 离线使用

首次在线打开后，service worker 会缓存静态文件。之后在同一浏览器中再次打开，基础页面和示例数据可以离线访问。

## 导出、导入和手动同步

进入“数据备份与同步”，点击“导出本地数据”，浏览器会下载 `erp-learning-backup-YYYY-MM-DD.json`。把电脑数据导入手机时，先在电脑导出 JSON 文件，再传到手机并在项目中导入。把手机数据导入电脑时反向操作。这是手动同步，不是自动云同步，本项目不会上传数据。

## 如何新增内容和练习

在术语库、ERP 字段解释器、指标口径库、运营动作库、业务场景分析库中填写新增表单并保存到本地。进入“业务练习题”可查看答案并标记已掌握。进入“指标计算器”选择计算类型、输入数字并计算。

## 后续可选扩展方向

可选扩展包括：增加更多术语和字段、增加更多脱敏业务场景、CSV/Excel 导入、本地备份、暗色模式、学习进度条、错题本、每日复习提醒、本地报表分析模板、本地数据看板。未来如果数据量变大，可以考虑 SQLite，但不是当前版本。

## 重要提醒

本项目不需要任何付费服务，不需要注册平台，不需要 API Key，不需要云服务。不要录入公司真实敏感数据，不要把公司订单、客户、销售、库存等敏感数据放入项目。


## GitHub Pages 免费部署说明

本项目是纯静态 PWA，可以放到 GitHub 仓库中，并通过 GitHub Pages 免费发布。发布后，手机和电脑都可以通过同一个网址访问这个学习工具。

注意事项：

- 代码仓库不要放 GitHub Personal Access Token。
- 初始 `data/*.json` 不要放公司真实敏感数据。
- GitHub Pages 只托管网页代码，不保存用户 token。
- 用户自己的学习数据保存在当前浏览器本地和你自己的 GitHub Gist 中。
- Gist 用于保存同步 JSON 文件 `erp-learning-sync.json`。

## GitHub Gist 同步说明

同步中心支持使用 GitHub Gist API 保存学习数据。你需要自己创建 GitHub Personal Access Token，并填写到每台设备的同步中心。

安全要求：

- 不要把 token 发给任何人。
- token 建议只给 Gist 写权限。
- token 只保存在当前设备浏览器本地。
- token 不会导出到本地备份 JSON。
- token 不会上传到 Gist。
- token 不应该写入 README、代码仓库或 data JSON。
- 如果担心安全，可以只使用本地导出 / 导入，不使用云同步。

## 如何创建 Gist

1. 打开 GitHub Gist 页面。
2. 新建一个 Gist。
3. 文件名填写 `erp-learning-sync.json`。
4. 文件内容可以先写 `{}`。
5. 保存后，从地址栏复制 Gist ID。

## 如何配置 Personal Access Token

1. 在 GitHub 设置中创建 Personal Access Token。
2. 权限只选择 Gist 相关权限。
3. 复制 token。
4. 回到本项目“同步中心”。
5. 填写 GitHub Personal Access Token、Gist ID、设备名称。
6. 点击“保存配置”。
7. 点击“测试连接”。

README 中不会提供 token 示例，避免误把 token 公开。

## 电脑到手机同步

1. 电脑打开 PWA。
2. 在同步中心填写 GitHub Token、Gist ID、设备名称。
3. 点击“普通同步”或“强制上传”。
4. 手机打开同一个 PWA 地址。
5. 手机也填写同一个 GitHub Token 和 Gist ID。
6. 手机点击“普通同步”或“强制拉取”。
7. 手机即可看到电脑的数据。

## 手机到电脑同步

1. 手机新增学习笔记或术语。
2. 手机点击“普通同步”。
3. 电脑点击“普通同步”。
4. 电脑即可看到手机新增内容。

## 普通同步、强制上传、强制拉取的区别

- 普通同步：先拉取云端，再合并本地和云端。按 `id` 判断同一条数据，按 `updatedAt` 保留较新的版本。推荐日常使用。
- 强制上传：当前设备数据覆盖 Gist 云端数据。适合你确定当前设备数据最完整时使用。
- 强制拉取：Gist 云端数据覆盖当前设备本地用户数据。适合新设备首次同步或本地数据已经不需要时使用。

删除操作采用软删除，会保留 `isDeleted` 和 `deletedAt`，避免一个设备删除后另一个设备又把旧数据同步回来。

## 多设备同步注意事项

- 尽量每次换设备前先同步。
- 如果不确定，以普通同步为主。
- 强制上传和强制拉取要谨慎。
- 不要多台设备同时大量编辑同一条内容。
- 如果出现冲突，看同步中心的冲突日志。

## 如何验证同步成功

1. 在电脑新增一条学习笔记。
2. 电脑点击“普通同步”。
3. 手机打开同一个 PWA，填写相同 Gist 配置。
4. 手机点击“普通同步”。
5. 如果手机能看到电脑新增的笔记，说明同步成功。

## 如何验证离线可用

1. 用本地服务器或 GitHub Pages 打开一次项目。
2. 等页面加载完成。
3. 断开网络。
4. 重新打开页面，检查首页、术语库、字段库、指标库、练习题和学习笔记是否可用。
5. 没有网络时，同步按钮会提示网络错误，但本地新增和本地备份仍可使用。

## 如何验证 PWA 安装

在支持 PWA 的浏览器中打开项目地址。如果浏览器支持安装，地址栏或菜单中会出现“安装应用”或“添加到主屏幕”。PWA 安装能力可能受浏览器和 HTTPS 限制，GitHub Pages 发布后更容易测试手机安装。

## 2.0 电商知识成长系统

这个项目现在不只是资料库，而是一个本地优先的电商知识成长系统。它帮助你从电商基础、商品 SKU、订单、库存、售后、运营、指标，一步一步学到业务分析思维。

### 它不做什么

- 不连接真实 ERP。
- 不抓真实平台数据。
- 不接 AI API。
- 不需要登录注册。
- 不需要数据库。
- 不使用付费服务。

### 怎么进入成长中心

打开项目后，点击左侧菜单里的“电商知识成长中心”。这里会显示当前学习阶段、今日任务、知识地图、阶段课程、业务案例、复习中心和成长记录。

### 怎么完成今日任务

进入“电商知识成长中心”，看“今日学习任务”。每天会自动给你一组课程、术语、ERP 字段、指标、运营动作、案例和练习题。学完后点击“完成今日任务”。如果今天这组不合适，可以点“换一组”。

### 怎么学习课程

在“阶段课程”里按当前阶段学习。每节课都包含通俗解释、业务例子、ERP 关联、运营关联、数据分析关联和常见误解。学完后点“记录完成”。

### 怎么标记掌握状态

每节课程和每个案例都有“掌握状态”。可以选“未学习”“学习中”“已掌握”“需要复习”。选“需要复习”的内容会自动进入复习中心。

### 怎么用复习中心

复习中心会集中显示你标记为“需要复习”的内容。复习后可以把状态改成“已掌握”。

### 怎么用案例学习

“业务案例训练”使用模拟电商场景，不包含真实公司数据。你可以按背景、数据现象、分析步骤和结论来练习业务分析思路。

### 怎么生成 ChatGPT 提问模板

课程和案例里有“复制提问模板”按钮。复制后可以打开 ChatGPT，把模板粘贴进去继续问。项目不会调用 OpenAI、DeepSeek 或任何付费 API，只是帮你生成问题文本。

### 联网增强中心有什么用

联网增强中心只做免费增强：检查当前网络状态、检查课程包是否能访问、检查版本文件、打开 ChatGPT、GitHub 项目和你的 Gist 同步入口。WiFi 和手机流量都可以用；离线时本地学习功能仍然能用。

### 怎么同步学习进度

如果已经配置 GitHub Gist，同步中心会把课程完成、案例完成、掌握状态、复习内容和成长记录一起同步。日常建议使用“普通同步”。不确定时不要点强制上传或强制拉取。
